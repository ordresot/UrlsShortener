# Сервис для создания коротких ссылок

Сервис для сокращения длинных URL-адресов с возможностью установки лимита переходов и автоматическим удалением по истечении срока жизни.

## Возможности

- **Создание коротких ссылок** - преобразование длинных URL в короткие формата `clck.ru/AbCdEf`
- **Уникальность для каждого пользователя** - даже одинаковые длинные URL дают разные короткие ссылки
- **Лимит переходов** - возможность задать максимальное количество переходов (например, 5, 10, 100)
- **Ограничение времени жизни** - ссылки автоматически удаляются через заданное время (по умолчанию 24 часа)
- **Идентификация пользователя** - каждый пользователь идентифицируется по уникальному UUID
- **Управление ссылками** - просмотр, редактирование лимита и удаление только своих ссылок
- **Автоматическое открытие в браузере** - при переходе по короткой ссылке

## Требования

- Java 8 или выше
- Операционная система: Windows, macOS или Linux
- Наличие браузера по умолчанию

## Установка и запуск

### 1. Клонирование репозитория
```bash
git clone https://github.com/yourusername/url-shortener.git
cd url-shortener
```

### 2. Компиляция проекта

```bash
# Создаем директорию для скомпилированных классов
mkdir -p out

# Компилируем все Java файлы
javac -d out $(find src/main/java -name "*.java")
```

### 3. Запуск приложения

```bash
# Переходим в директорию с классами
cd out

# Запускаем главный класс
java com.urlshortener.Main
```

## Конфигурация

При первом запуске автоматически создается файл конфигурации `config/urlshortener.properties`:

```properties
# Время жизни ссылки в часах
# По умолчанию: 24 часа
link.lifetime.hours=24

# Базовый URL для коротких ссылок
# По умолчанию: clck.ru/
base.url=clck.ru/

# Длина генерируемого кода (рекомендуется 4-10 символов)
# По умолчанию: 6
short.code.length=6
```

## Использование

### Первый запуск

При запуске система предложит два варианта идентификации:

- **У меня уже есть UUID** - если вы уже пользовались сервисом и сохранили свой UUID
- **Я новый пользователь** - UUID будет сгенерирован автоматически при создании первой ссылки

#### Создание первой ссылки:

1. Выберите пункт меню **1** (Создать короткую ссылку)
2. Введите длинный URL, который хотите сократить
3. Укажите лимит переходов (например, 5, 10 или 100)
4. **ВАЖНО**: Сохраните сгенерированный UUID! Он понадобится для доступа к вашим ссылкам в будущем

### Основные операции

| Действие | Последовательность |
|----------|-------------------|
| **Создание ссылки** | Меню → **1** → ввод URL → ввод лимита переходов |
| **Переход по ссылке** | Меню → **2** → ввод короткой ссылки (например, `clck.ru/AbCdEf`) |
| **Просмотр своих ссылок** | Меню → **3** → отображение всех ваших ссылок с деталями |
| **Изменение лимита** | Меню → **4** → ввод кода ссылки → новый лимит переходов |
| **Удаление ссылки** | Меню → **5** → ввод кода ссылки для удаления |
| **Сменить пользователя** | Меню → **6** → выход в меню идентификации |

## Тестирование

### 1. Создание ссылки
- Введите URL: `https://www.google.com`
- Лимит: `5`
- **Ожидаемый результат**: Должна создаться ссылка вида `clck.ru/AbCdEf`

### 2. Переход по ссылке
- Введите созданную ссылку
- **Ожидаемый результат**: Должен открыться браузер и открыть вкладку с указанной ссылкой

### 3. Исчерпание лимита
- Перейдите по ссылке 5 раз
- На 6-й раз должно появиться сообщение об исчерпании лимита
- **Ожидаемый результат**: Сообщение `"Ошибка: Лимит переходов исчерпан"`

### 4. Смена пользователя
- Введите другой UUID (или выберите создание нового пользователя)
- Создайте ссылку на тот же URL `https://www.google.com`
- **Ожидаемый результат**: Должен получиться другой короткий код (отличный от `AbCdEf`)
